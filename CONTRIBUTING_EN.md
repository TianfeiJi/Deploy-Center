## Commit Message Convention

Please use the Conventional Commits format:

```
<type>: <short description>
```

### Common Types:

| Type     | Description                          |
|----------|--------------------------------------|
| feat     | New feature or module                |
| fix      | Bug fix                              |
| refactor | Code refactoring (no functional change) |
| docs     | Documentation updates                |
| style    | Code style changes (e.g. spaces, indentation) |
| chore    | Build system / scaffold / dependency updates |
| test     | Add or modify tests                  |
| perf     | Performance improvement              |

### Examples:

```bash
feat: implement scheduled deployment feature
fix: fix issue with xxx
docs: add documentation for xxx
```

---

## Recommended Collaboration Process

1. Fork this project and create a new branch:

```bash
git checkout -b feat/your-feature-name
```

2. Follow the commit message convention when committing code

3. When submitting a PR:

- Provide a clear title and description
- Include screenshots / reference links / reproduction steps if applicable
- Link related issues if any

---

## Key Points to Keep in Mind

- Maintain clean code and consistent naming
- Avoid large mixed commits (prefer small incremental commits)
- Keep Chinese documentation in sync (if applicable)

---

If you have any questions or suggestions, feel free to open an Issue or start a Discussion!
